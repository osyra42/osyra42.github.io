How to Use ComfyUI

# 🎨 How to Use ComfyUI ::toc:: ## 🖌️ What is ComfyUI? [ComfyUI](https://github.com/comfyanonymous/ComfyUI) is a node-based interface for Stable Diffusion and other generative AI models. Instead of a single text box, you build your image generation pipeline by wiring nodes together - similar to Blender's shader nodes or Unreal's Blueprints. **✅ Why use ComfyUI?** - **Visual workflow** - See exactly how your generation pipeline works - **Reusable workflows** - Save and share complex setups as JSON files - **Fine control** - Adjust every parameter in the generation process - **Efficient** - Only re-runs changed parts of your workflow - **Extensible** - Huge library of custom nodes from the community --- ## 🚀 Getting Started The fastest way to get running on Windows with an NVIDIA GPU: 1. Download the **portable build** from the [ComfyUI releases page](https://github.com/comfyanonymous/ComfyUI/releases). 2. Extract the `.7z` somewhere with plenty of space (models are large). 3. Run `run_nvidia_gpu.bat` (use `run_cpu.bat` only if you have no compatible GPU - it's very slow). 4. ComfyUI opens in your browser at `http://127.0.0.1:8188`. For full instructions and other platforms, see the [official ComfyUI documentation](https://docs.comfy.org/). #### 📁 Where Each File Goes Drop your downloaded models into the matching folder under `ComfyUI/models/`: | File type | Folder | | --- | --- | | Checkpoints | `models/checkpoints/` | | LoRAs | `models/loras/` | | VAE | `models/vae/` | | Upscale models | `models/upscale_models/` | Restart ComfyUI (or hit **Refresh** in the browser) after adding files so they show up in the node dropdowns. #### 🧩 ComfyUI Manager Install [ComfyUI Manager](https://github.com/ltdrdata/ComfyUI-Manager) early - it's the easiest way to install custom nodes, update ComfyUI, and auto-fetch any missing nodes when you load someone else's workflow. Follow its install steps, then look for the **Manager** button in the ComfyUI menu. #### 🖥️ Hardware / VRAM SDXL models (like the checkpoint below) want roughly **8 GB of VRAM** to run comfortably at 1024x1024. If you're tight on memory, launch with one of these flags: - `--medvram` - balances speed and memory use (good first choice) - `--lowvram` - for cards with very little VRAM; slower but gets you running Add the flag inside the `.bat` file or on the command line. --- ## ⚙️ My Default Settings These are what I use for most of my generations. Tweak them to suit your hardware and what you're after. **Model & LoRAs:** - **Checkpoint:** [ChalicePDXL_v0.4](https://mega.nz/file/XdEGgKKD#69LIoYJyI6qYEmkuY5U07SxO2d9PsEbZnD5WmYiIUnE) - **LoRAs:** [HandFineTuning_XL](https://civitai.com/models/278497/hand-fine-tuning?modelVersionId=358694) and [add-detail-xl](https://huggingface.co/LyliaEngine/add-detail-xl/tree/main), both at strength **~1.0** - **CLIP Skip:** -2 - **VAE:** [sdxl_vae](https://huggingface.co/stabilityai/sdxl-vae/tree/main) **Generation Settings:** - **Width/Height:** 1024 x 1024 - **Steps:** 24-30 - **CFG Scale:** 7.0 - **Sampler:** dpmpp_2m - **Scheduler:** karras **Upscaling:** - **Model:** [RealESRGAN_x4plus_anime](https://civitai.com/models/147821/realesrganx4plus-anime-6b) For more on samplers and schedulers, see the [ComfyUI node documentation](https://docs.comfy.org/built-in-nodes/overview). --- ## 🔗 Basic Workflow When you first open ComfyUI it loads a default text-to-image graph. If you ever need to rebuild it, this is the core chain - each node feeds the next: 1. **Load Checkpoint** - loads your model. Outputs three things: MODEL, CLIP, and VAE. 2. **CLIP Text Encode (Prompt) ×2** - one for your positive prompt, one for your negative prompt. Both take the CLIP output from the checkpoint. 3. **KSampler** - the heart of the graph. Wire in MODEL (from checkpoint), the positive and negative conditioning (from the two CLIP encoders), and a **Empty Latent Image** for the canvas size (1024x1024). This is where steps, CFG, sampler, and scheduler from the settings above live. 4. **VAE Decode** - turns the KSampler's latent output into an actual image, using the VAE from the checkpoint. 5. **Save Image** - writes the decoded image to `ComfyUI/output/`. To add a LoRA, insert a **Load LoRA** node between the checkpoint and the rest: feed it MODEL + CLIP, and route its outputs onward. For upscaling, add an **Upscale Image (using Model)** node after the VAE Decode and point it at your upscale model. ComfyUI only re-runs the nodes that changed, so tweaking just the prompt won't re-run the whole graph. --- ## 🚫 Negative Prompt A solid negative prompt cuts out a lot of the usual artifacts. Here's the one I use: ``` fused mouth, blurry mouth, ugly mouth, fused cheek, blurry cheek, ugly cheek, too many fingers, bad feet, deformed, bad body, malformed limbs, extra limb, extra arms, extra legs, acnes, long body, ugly, bad hands, lowres, bad anatomy, text, error, missing fingers, extra digit, fewer digits, cropped, worst quality, low quality, normal quality, jpeg artifacts, signature, watermark, username, blurry, (3d:1.3), broken limb, incorrect limb, fusion finger, lost finger, multiple fingers, multiple digits, fusion hand, lost leg, fused leg, multiple legs, jpeg, forehead marks, forehead jewels, censored ``` **Key things this addresses:** - **Hands/fingers** - Common problem area for AI generation - **Body proportions** - Prevents extra limbs and deformations - **Quality tags** - Filters out low-quality outputs - **Artifacts** - Removes watermarks, signatures, and compression artifacts - **Style control** - (3d:1.3) reduces unwanted 3D render look --- ## 🎨 Example: Osyra Character Here's an example prompt for generating my character Osyra: ``` score_9, score_8_up, score_7_up, best quality, 4k, masterpiece, digital art, 1girl, solo, (brown skin:1.7), (black hair:1.8), long hair, wolf ears, wolf tail, red eyes, adult woman, round glasses, swept bangs, brown jacket unzipped, jacket with a hood, light yellow shirt, and blue pants, ``` **Breaking down the prompt:** - **Quality tags** - score_9, score_8_up, score_7_up are used by Pony-based models to filter for high-quality outputs. Additional tags like "best quality, 4k, masterpiece" reinforce this. - **Character basics** - 1girl, solo establishes a single female character. The weighted tags (brown skin:1.7) and (black hair:1.8) use higher emphasis to ensure these features appear consistently. - **Physical features** - wolf ears, wolf tail, red eyes, round glasses, swept bangs define the character's distinct look. - **Outfit** - Clothing is described last: brown hooded jacket (unzipped), light yellow shirt underneath, and blue pants. **Prompt structure tips:** - **Weight syntax** - Use (tag:1.5) to increase emphasis, higher = stronger - **Order matters** - Put important features early in the prompt - **Commas** - Separate distinct concepts with commas - **Line breaks** - Group related concepts together for readability --- ## 🎭 Character Prompts These are tested prompts for my original characters. Make sure a VAE is loaded, and try adding the `--no-half-vae` launch flag. - **Out of memory (OOM)** - drop to a smaller resolution, or launch with `--medvram` / `--lowvram`. Tiled VAE decode also helps on large images. - **"Missing node" / red nodes** when loading a workflow - open **ComfyUI Manager → Install Missing Custom Nodes**, then restart. - **Models not showing in dropdowns** - confirm the file is in the right `models/` subfolder, then hit **Refresh** or restart ComfyUI. --- ## 📚 Resources - [ComfyUI GitHub](https://github.com/comfyanonymous/ComfyUI) - Source code and releases - [ComfyUI Documentation](https://docs.comfy.org/) - Official guides and tutorials - [CivitAI](https://civitai.com/) - Models, LoRAs, and community workflows - [ComfyUI Manager](https://github.com/ltdrdata/ComfyUI-Manager) - Essential extension for managing custom nodes **Models I use:** - [ChalicePDXL_v0.4](https://mega.nz/file/XdEGgKKD#69LIoYJyI6qYEmkuY5U07SxO2d9PsEbZnD5WmYiIUnE) - Checkpoint - [HandFineTuning_XL](https://civitai.com/models/278497/hand-fine-tuning?modelVersionId=358694) - LoRA - [add-detail-xl](https://huggingface.co/LyliaEngine/add-detail-xl/tree/main) - LoRA - [RealESRGAN_x4plus_anime](https://civitai.com/models/147821/realesrganx4plus-anime-6b) - Upscale model - [sdxl_vae](https://huggingface.co/stabilityai/sdxl-vae/tree/main) - VAE --- ::signature::
