# 🚀 Zen Launcher Zen Launcher is a small launcher script that sits next to a program and runs a sanity check before starting it. Users double-click one file — `zen_launcher.bat` on Windows, `zen_launcher.sh` on Mac and Linux — and the launcher walks through the checks before handing off to the actual program. --- ## What It Does Zen Launcher runs through three stages before launching the main file: **1. Environment Check** Verifies the runtime the program needs is actually installed — Python, Java, Node.js, .NET, Ruby, PHP, whatever the project depends on. If something's missing, the user gets a clear message pointing at the specific missing piece, not a stack trace. **2. Dependency Check** Reads the main file and confirms the dependencies it declares are present (e.g. packages from `requirements.txt`, libraries from `package.json`). Surfaces missing packages before they cause a crash mid-run. **3. Launch** Once the checks pass, Zen Launcher runs the main file. It also sets the terminal window title and any environment variables the program needs at startup, so the launched process inherits a clean, named context. --- ## Why It Exists Distributing a small program to non-technical users is brittle. They hit "command not found" errors, miss a runtime version requirement, or don't know to install dependencies. The program crashes mid-run with an opaque traceback. Zen Launcher front-loads those failures into a single readable check phase. The user sees something like *"Python is required to run this program"* instead of a wall of errors. The launcher is the only thing they ever click — the actual program file (`app.py`, `game.jar`, `bot.js`) sits next to it and never gets touched directly. --- ## How It Looks A typical bundle: ``` my_project/ ├─ zen_launcher.bat (user double-clicks this on Windows) ├─ zen_launcher.sh (user double-clicks this on Mac/Linux) ├─ app.py (the actual program) ├─ requirements.txt └─ ... ``` The launcher scripts are short, hand-editable shell/batch files. There's no compiled binary or installer to manage — just two text files that anyone can read, audit, or tweak. --- ## How I Use It Every program I distribute ships with Zen Launcher next to it.
